A은행 소득공제 업무
- 업무설명
    - 본 업무는 A은행에서 발생한 카드 승인 및 매입 데이터, 가맹점 정보, 고객 정보 등을 수집한 뒤 소득공제 대상 여부를 판별하고, 정제된 결과를 내부 관리 시스템과 외부 제출 파일에 반영하는 일련의 처리 과정으로 구성된다.
    주요 처리 대상은 체크카드, 신용카드, 현금성 거래 중 소득공제 대상이 되는 거래이며, 거래 취소, 오류 데이터, 대상 제외 업종 등은 별도 기준에 따라 제외 또는 보정 처리한다.
- 배치작업
    1단계 : 데이터 적재(병렬실행)
        1. BATCH_01_CARD_APPROVAL_LOAD
            카드 승인 데이터 수집 및 적재
        2. BATCH_02_CARD_PURCHASE_LOAD
            카드 매입 데이터 수집 및 적재
        3. BATCH_03_CUSTOMER_INFO_LOAD
            고객 기본정보 적재
        4. BATCH_04_MERCHANT_INFO_LOAD
            가맹점 정보 및 업종코드 적재
        5.  BATCH_05_CANCEL_DATA_LOAD
            취소 및 정정 거래 데이터 적재
    2단계 : 소득공제 대상 처리 배치 (1단계 완료 후 실행)
        1. BATCH_06_DEDUCTION_TARGET_FILTER
            소득공제 대상 거래 판별
            제외 업종 및 비대상 거래 제거
        2. BATCH_07_DEDUCTION_AGGREGATION
            고객별 소득공제 금액 집계
            월별/연간 기준 데이터 생성
    3단계 : 최종 결과 생성 배치 (2단계 배치 완료 후 실행)
        1. BATCH_08_DEDUCTION_FILE_GENERATION
            대외 제출용 소득공제 파일 생성
            내부 조회 시스템 반영 데이터 생성
- 테이블 관계도
    1단계 : 원천 데이터 테이블 (적재 영역)
        TB_CARD_APPROVAL
        TB_CARD_PURCHASE
        TB_CUSTOMER_INFO
        TB_MERCHANT_INFO
        TB_CANCEL_DATA

    2단계 : 중간 처리 테이블
        TB_DEDUCTION_TARGET
            소득공제 대상 거래 필터링 결과 저장
        TB_DEDUCTION_EXCLUDE
            제외 업종 및 비대상 거래 관리
        TB_DEDUCTION_AGG
            고객 기준 소득공제 금액 집계 데이터

    3단계 : 결과 테이블
        TB_DEDUCTION_RESULT
            고객별 최종 소득공제 결과
        TB_DEDUCTION_FILE
            대외 제출 파일 생성 데이터
        TB_DEDUCTION_LOG
            배치 수행 로그 및 오류 내역

개발환경 구축
- 개발환경 개요
    - 본 문서는 A은행 청구업무 시스템의 유지보수 및 개발을 위한 Java / Eclipse / Tomcat 기반 개발환경 구축 절차를 정리한 문서이다.
    - 청구업무는 주로 웹 관리 화면, 배치 실행, DB 조회 및 운영성 점검 기능으로 구성되며, 개발자는 동일한 표준 환경을 구성하여 프로그램 수정, 테스트, 배포 준비를 수행하여야 한다.
    - 개발환경은 Java JDK, Eclipse IDE, Apache Tomcat, DB 접속 환경, 형상관리 도구, 공통 라이브러리로 구성된다.


- 개발환경 구성 요소
    1. 운영체제
        Windows 10 또는 Windows 11
    2. JDK
        Java JDK 1.8
    3. IDE
        Eclipse IDE for Enterprise Java Developers
    4. WAS
        Apache Tomcat 8.5 또는 9.0
    5. DB Tool
        DBeaver 또는 SQL Developer
    6. 형상관리
        Git 또는 사내 SVN
    7. 빌드도구
        Maven 또는 Gradle
    8. 기타
        JDBC Driver, 로그 확인 도구, 배치 실행 스크립트

- 개발환경 설치 경로 예시
    1. JDK 설치 경로
        C:\dev\Java\jdk1.8.0_xxx
    2. Eclipse 설치 경로
        C:\dev\eclipse
    3. Tomcat 설치 경로
        C:\dev\tomcat9
    4. Workspace 경로
        C:\workspace\a-bank-billing
    5. 공통 라이브러리 경로
        C:\dev\libs
    6. 배치 스크립트 경로
        C:\batch\a-bank-billing

- 1단계 : JDK 설치
    1. Oracle JDK 또는 OpenJDK 1.8 설치파일을 준비한다.
    2. 지정 경로에 JDK를 설치한다.
    3. 시스템 환경변수를 등록한다.
        JAVA_HOME = C:\dev\Java\jdk1.8.0_xxx
        PATH = %JAVA_HOME%\bin 추가
    4. 명령 프롬프트에서 버전을 확인한다.
        java -version
        javac -version
    5. 정상적으로 버전이 출력되면 설치 완료로 판단한다.

- 2단계 : Eclipse 설치
    1. Eclipse IDE for Enterprise Java Developers 버전을 다운로드한다.
    2. 압축 해제 또는 설치 후 실행한다.
    3. 최초 실행 시 Workspace 경로를 지정한다.
        C:\workspace\a-bank-billing
    4. 인코딩 설정을 UTF-8로 변경한다.
        Window > Preferences > General > Workspace > Text file encoding > UTF-8
    5. 서버 및 웹 개발 관련 플러그인이 포함되어 있는지 확인한다.
    6. 필요 시 Lombok, Spring Tools, Git 플러그인을 추가 설치한다.

- 3단계 : Tomcat 설치
    1. Apache Tomcat 8.5 또는 9.0 버전을 다운로드한다.
    2. 압축을 해제하여 지정 경로에 배치한다.
        C:\dev\tomcat9
    3. conf, logs, webapps, temp, work 폴더가 정상 생성되었는지 확인한다.
    4. startup.bat 실행으로 서버 기동 여부를 점검한다.
    5. 브라우저에서 아래 주소로 정상 페이지 확인한다.
        http://localhost:8080
    6. 포트 충돌 시 server.xml에서 포트를 변경한다.
        예: 8080 -> 8081

- 장애현황 조회

    - 설명
        배치 수행 중 발생한 오류 및 장애 내역을 조회하는 기능으로, 배치명 기준으로 오류 메시지 및 발생시간을 확인한다.

    - 조회 기준
        조회일자 : 당일 기준
        상태 : ERROR 또는 FAIL

    - 사용 테이블
        TB_DEDUCTION_LOG (소득공제)
        TB_BILLING_LOG (청구)

    - 주요 컬럼
        JOB_NAME        : 배치명
        STATUS          : SUCCESS / FAIL
        ERROR_MESSAGE   : 오류 내용
        ERROR_CODE      : 오류 코드
        START_TIME      : 시작시간
        END_TIME        : 종료시간

    - 조회 예시
        SELECT
            JOB_NAME,
            STATUS,
            ERROR_MESSAGE,
            START_TIME
        FROM TB_BILLING_LOG
        WHERE STATUS = 'FAIL'
          AND START_TIME >= SYSDATE

    - 응답 형태
        배치명 / 오류내용 / 발생시간 기준으로 요약하여 제공

- 청구 이용내역 조회

    - 설명
        고객의 월별 카드 이용 내역 및 청구 금액을 조회하는 기능으로, 이번달 기준 청구 대상 거래 및 이용내역서를 확인한다.

    - 조회 기준
        조회기간 : 당월 기준
        고객조건 : 고객 ID 또는 계좌번호 기준
        상태 : 정상 승인 거래 기준 (취소 거래 제외 또는 별도 표시)

    - 사용 테이블
        TB_CARD_APPROVAL
        TB_CARD_PURCHASE
        TB_BILLING_TARGET
        TB_BILLING_CALC
        TB_BILLING_RESULT
        TB_BILLING_STATEMENT

    - 주요 컬럼
        CUSTOMER_ID     : 고객 ID
        CARD_NO         : 카드번호
        APPROVAL_DATE   : 승인일자
        APPROVAL_AMOUNT : 승인금액
        MERCHANT_NAME   : 가맹점명
        INSTALLMENT     : 할부 개월수
        BILLING_AMOUNT  : 청구금액
        BILLING_MONTH   : 청구월

    - 조회 예시
        SELECT
            CUSTOMER_ID,
            CARD_NO,
            APPROVAL_DATE,
            MERCHANT_NAME,
            APPROVAL_AMOUNT,
            BILLING_AMOUNT
        FROM TB_BILLING_STATEMENT
        WHERE BILLING_MONTH = TO_CHAR(SYSDATE, 'YYYYMM')

    - 응답 형태
        고객별 이용내역을 기준으로 거래일자, 가맹점, 이용금액, 청구금액을 리스트 형태로 제공


- 업무요약

    A은행 소득공제 업무
        카드 승인 및 매입 데이터를 기반으로 소득공제 대상 거래를 판별하고 집계 및 대외 제출 파일을 생성하는 배치 중심 업무

    B증권 소득공제 업무
        계좌 기반 금융거래를 포함하여 소득공제 대상 거래를 판별하고 고객 및 계좌 기준으로 집계 처리하는 업무

    A은행 청구 업무
        카드 이용 데이터를 기반으로 고객별 청구 금액을 산정하고 청구서를 생성하는 업무

    B증권 청구 업무
        카드 및 계좌 기반 거래를 통합하여 고객별 청구 금액을 계산하고 청구서를 생성하는 업무

[업무 역할 정의]
- 본 업무는 소득공제 및 청구 업무의 개발, 운영, 유지보수를 수행하는 업무이다.
- 주요 수행 내용
  - 배치 개발 및 운영
  - 장애 대응 및 원인 분석
  - 데이터 조회 및 검증
  - 대외 파일 생성 및 검증
  - 고객사 대응 및 운영 지원

[업무 수행 순서]
1. 개발환경 구축 (JDK, Eclipse, Tomcat, DB Tool 설치)
2. 소스 코드 및 형상관리 연결
3. DB 접속 및 주요 테이블 확인
4. 배치 흐름 및 업무 프로세스 이해
5. 테이블 관계 및 데이터 흐름 이해
6. 운영 점검 및 장애 대응 방법 숙지

[장애 대응 절차]
1. 장애 발생 배치 확인
2. TB_DEDUCTION_LOG / TB_BILLING_LOG 조회
3. 오류 메시지 및 ERROR_CODE 확인
4. 관련 테이블 데이터 정합성 확인
5. 재처리 여부 판단
6. 필요 시 담당자 및 고객사 공유

[배치-테이블 매핑]
- BATCH_01~05 : 원천 데이터 → TB_CARD_APPROVAL, TB_CARD_PURCHASE 등 적재
- BATCH_06 : TB_DEDUCTION_TARGET / TB_BILLING_TARGET 생성
- BATCH_07 : 집계 테이블 생성 (TB_DEDUCTION_AGG / TB_BILLING_CALC)
- BATCH_08 : 결과 테이블 생성 (TB_DEDUCTION_RESULT / TB_BILLING_RESULT)

[조회 파라미터 정의]
- CUSTOMER_ID : 고객 식별자
- ACCOUNT_NO : 계좌번호
- CARD_NO : 카드번호
- BILLING_MONTH : 조회 기준 월
- DOMAIN : 소득공제 / 청구 구분
- CUSTOMER_TYPE : 고객사 구분 (A은행, B증권 등)
